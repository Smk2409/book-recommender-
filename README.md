# The Reading Room — Book Recommendation System (ML + Flask)

A mini-project that recommends books using two complementary machine
learning approaches, served through a Flask web app.

## Problem statement

With thousands of books across genres, authors, and publishers, readers
struggle to discover titles that match their taste using search alone. This
project builds a web-based recommendation system that suggests books based
on overall popularity and on similarity to a book the user already likes.

## Dataset

[goodbooks-10k](https://github.com/zygmuntz/goodbooks-10k) — 10,000 books
and ~6 million user ratings (1–5 stars), a cleaned, well-structured
alternative to the classic Book-Crossing dataset. Included in `data/`:

- `books.csv` — book metadata (title, author, average rating, cover image, etc.)
- `ratings.csv` — `user_id, book_id, rating` triples

## Approach

Two models, both pre-computed offline and loaded by the Flask app at
runtime (no retraining on each request):

### 1. Popularity-based recommender
Ranks books with a weighted-rating formula (the same idea IMDB uses for its
Top 250), so a book with a handful of 5-star ratings can't outrank one
loved by tens of thousands of readers:

```
WR = (v / (v + m)) * R + (m / (v + m)) * C
```
- `R` = the book's average rating
- `v` = number of ratings it has received
- `m` = minimum ratings threshold (75th percentile of ratings_count)
- `C` = mean rating across the whole catalogue

This powers the "Top Picks" home page.

### 2. Item-based collaborative filtering
Builds a `book x user` pivot table of ratings (restricted to active users
and reasonably-rated books, ~6,000 books x ~4,200 users, to keep the matrix
a manageable size) and computes cosine similarity between books based on
the pattern of how users rated them. Two books end up "close" together when
the same readers tend to rate them similarly — this is what powers "Find
Similar Books."

## Project structure

```
book-recommender/

├── model_builder.py        # Builds & pickles both models from the CSVs
├── requirements.txt
├── data/
│   ├── books.csv

├── models/                 # Generated by model_builder.py
```

## Running it

```bash
pip install -r requirements.txt

# Build the models (only needs to be run once, or whenever the CSVs change)
python model_builder.py

# Start the app
python app.py
```

Then open **http://127.0.0.1:5000** in a browser.

- `/` — the popularity-based "Top Picks" page
- `/recommend` — type or pick a book title to get 5 similar recommendations

## Possible extensions

- Swap the fixed popularity/CF split for a hybrid score that blends both.
- Add genre/tag-based content filtering (goodbooks-10k ships a `tags.csv`
  and `book_tags.csv` for this).
- Add real user accounts so ratings feed back into the similarity matrix.
- Deploy with a production WSGI server (gunicorn) instead of the Flask dev server.
